# V5 · Detalle de óptica

**URL:** `<vista>?optica=<place_id>` · **Roles:** todos (lectura) · admin (edición vía overrides) · **Patrón:** drawer lateral derecho

## V5.1 Propósito
Consulta completa de los datos de una óptica sin romper el contexto del listado/mapa. Para admin: edición vía `opticas_overrides` (correcciones manuales sobre los campos read-only de Google/CPV), edición directa de `opticas_app_data` (show_campañas, notas).

## V5.2 Datos consumidos

Para el `place_id` solicitado:
- Fila completa de `opticas_google` (con overrides aplicados on top)
- Fila vinculada de `opticas_cpv` si `is_client = true` (con overrides aplicados)
- Fila de `opticas_app_data` (campos gestionados desde la app)
- Lista de overrides activos del `opticas_overrides` log para este `place_id`
- Lista de cambios de las últimas 10 entradas del log

## V5.3 Layout

```
┌────────────────────────────────────────────┐  drawer 480px (560 ≥1600)
│  Óptica San Carlos                    [X]  │  header sticky
│  ★ Cliente · Match confirmado · ✎ 2        │
│  C/ Mayor 23, 28013 Madrid                 │
│ ──────────────────────────────────────────│
│ [Google] [CPV] [Cambios] [App] [Estado]   │  tabs (5)
│ ──────────────────────────────────────────│
│         (CONTENIDO DEL TAB)                │  scroll
```

Header ~120px, tabs 44px, footer 60px (admin), contenido scroll.

# A3 · Revisión de datos

**URL:** `/admin/revision` · **Icono:** `warning-circle`

## A3.1 Propósito
Bandeja de matching post-sync. Vista crítica del admin. La semántica es: decidir si un registro `opticas_cpv` representa la misma óptica que un registro `opticas_google` (crear FK `place_id_fk`) o no.

**Cambio respecto a versiones previas:** A3 NO consolida campos. Solo crea/no crea vínculos FK. Para correcciones de campos existe `[✎ Corregir dato]` que abre el modal de override.

## A3.2 Datos
Registros candidatos: `opticas_cpv` con `place_id_fk` nulo o conflictivo. Por cada uno: lista de candidatos posibles de `opticas_google` con score de similitud, motivo del conflicto.

Estados:
- `conflicto`: `opticas_cpv` con 1+ candidatos de Google con score entre 60% y 85%
- `solo_cpv`: `opticas_cpv` sin candidatos viables (<60% en todos)
- `no_encontrado`: `opticas_google` previo desaparecido en última sync (flag en `opticas_app_data`)

## A3.4 Componentes (modo rápido)

**Tabs de estado** con contadores. **Focus card** (`card-accent`, una visible).

### Para tipo `conflicto`

Header "Conflicto N de M" + métricas (similitud, distancia, CP, ciudad). Body 2 columnas side-by-side: `opticas_google` izquierda, `opticas_cpv` derecha. Si hay más de un candidato en Google: dropdown "Candidato 1 de 3 ▼" arriba de la columna izquierda para alternar.

**Tres botones de decisión:**

| Botón | Estilo | Acción técnica |
|---|---|---|
| ✓ Es la misma · Vincular | primary | `UPDATE opticas_cpv SET place_id_fk = <place_id_candidato> WHERE CODIGO = ?`. Marca `match_confirmado`. Registra evento `vinculo_creado` en histórico. Pasa al siguiente. |
| ✗ Son distintas | ghost | Marca el candidato propuesto como descartado. Si era el único, el CPV queda en `solo_cpv`. Si quedan más candidatos, muestra el siguiente automáticamente. |
| ✎ Corregir datos | subtle | Abre modal "Corregir dato" (mismo que V5) sobre el campo que el admin elija. NO crea vínculo. Sirve para limpiar datos antes de decidir vincular o no. |

Saltar este: mueve al final de la cola.

### Para tipo `solo_cpv`

Header "Registro Salesforce sin coincidencia". 1 columna (solo CPV). Texto explicativo: "Existe en Salesforce pero no encontramos coincidencia en Google Maps. Puede ser óptica cerrada, no listada aún, o nombre/dirección demasiado distintos."

**Acciones:**

| Botón | Acción |
|---|---|
| 🔍 Buscar manualmente | Abre un modal con buscador para encontrar el `place_id` correcto en `opticas_google` (por nombre, dirección o place_id pegado). Al confirmar: crea vínculo FK. |
| ✗ Marcar como cerrada | Inserta override en `opticas_overrides` marcando estado como cerrada. CPV queda excluido de listados. |
| Mantener pendiente | Pasa al siguiente sin acción. |

### Para tipo `no_encontrado`

Header "Registro previo desaparecido". Datos previos del `opticas_google` que ya no aparece en la última sync.

**Acciones:**

| Botón | Acción |
|---|---|
| ✓ Mantener | Quita el flag de `no_encontrado` (Google la perdió por error, esperamos próxima sync). |
| ✗ Marcar como cerrada | Insert override de `business_status` = closed. |
| 🔍 Investigar | Abre modal para vincular manualmente a otro `place_id` (la óptica puede haberse re-listado bajo nuevo place_id). |

## A3.5 Tabla "Todos los pendientes"

Listado completo. Click en fila → esa pasa a focus card. Columnas: Tipo (pill), Nombre, Motivo, Detectado el, ⋯.

## A3.6 Flujos

**Secuencial:** entra con 47 → focus card conflicto 1 → decide en 3-10s → procesa y avanza → indicador "N de 47" → al terminar empty celebratorio.

**Skip:** "Saltar este" mueve al final de la cola.

**Deshacer:** toast post-decisión 10s "Vinculado con éxito. [Deshacer]".
