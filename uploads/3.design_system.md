# 3. Design system

Documento de referencia para diseño e implementación. Define tokens, componentes, patrones y reglas de uso. Construido sobre el **Newno Design System v1.1** (2026-05-21) con integración del logo CooperVision para el branding compartido.

**Output del diseño:** prototipo HTML navegable generado por Claude Design.
**Stack final del equipo técnico:** React + Vite + TypeScript · Bootstrap (CSS base) + tokens DS por encima · Iconoir (`iconoir-react`).
**Iconos en el prototipo HTML:** Iconoir vía web component `<iconify-icon icon="iconoir:...">`. Mismo set, distinto wrapper.

---

## 1. Principios

1. **Una vista, un acento.** El color de marca Newno (`#C5E817`) es un acento, no un protagonista. Aparece en máximo 1-2 elementos clave por pantalla (CTA principal, métrica destacada, ítem activo en sidebar). Si lo ves en más de dos sitios a la vez, sobra.
2. **Borde antes que sombra. Y aquí, doblemente.** Fondo de página y tarjetas son ambos blancos (#FFFFFF). La separación visual se hace con **bordes** (`--line` #C5C8CC), no con color de fondo. El borde es el que estructura. Las sombras son refuerzo sutil.
3. **Densidad cuando toca, aire cuando aporta.** Dashboards y BI con aire generoso. Tablas, listados y administración con densidad alta.
4. **Tipografía como jerarquía.** El peso y el tracking hacen la mayor parte del trabajo.
5. **Estados de datos siempre presentes.** Loading, empty, error y sin permisos están diseñados, no improvisados.
6. **Sin emojis como iconografía.** Iconoir, monocromo, hereda el color del texto.
7. **Branding compartido.** Newno construye, CooperVision usa. Convivencia explícita en zonas de marca.

**Diferencia clave respecto a un dashboard convencional:** fondo y card son ambos `#FFFFFF`. La interfaz se estructura por bordes. Usa `--line` para el borde estructural de cards y contenedores, y `--line-2` para separadores internos sutiles.

**Reglas del acento:**
- ✅ CTA primario, ítem activo en sidebar, marcador "cliente" en mapa, KPI destacado (uno solo), badge "Nuevo".
- ❌ Texto largo, fondos grandes, múltiples KPIs simultáneos, decoración.
- Texto sobre acento sólido: siempre `--accent-ink` (#282A2D). Nunca blanco.

**Contraste:** AA (4.5:1) para texto.

### 2.6 Transiciones

```css

```

Hover: fast. Drawers, modales: base. Skeleton shimmer: slow. Sin animaciones >240ms. Sin bounces.

## 5. Layout fundamental

### 5.1 Resolución objetivo

- Desktop-first. Target 1440×900. Mínimo soportado 1280×720.
- Sin responsive móvil en MVP. En <1280px: mensaje "Usa esta aplicación en un escritorio".

### 5.2 Estructura macro

```
┌───────────────────────────────────────────────────────────────┐
│  [Toggle rol]                                    [Avatar ▼]   │  topbar (prototype-only)
├───────────────────────────────────────────────────────────────┤
│  ┌────┐  ┌──────────────────────────────────────────────┐    │
│  │ 64 │  │  Padding: space-8 horizontal, space-6 vert.  │    │
│  │ px │  │  Max-width interno 1440px (auto margins)     │    │
│  │rail│  │                                              │    │
│  └────┘  └──────────────────────────────────────────────┘    │
└───────────────────────────────────────────────────────────────┘
```

- Sidebar 64px ancho fijo, sticky top:0, altura 100vh, fondo `--card` con `border-right: 1px solid --line`.
- Área principal: ancho `100% - 64px`, scroll vertical interno, padding `space-6 space-8`.

### 5.3 Grid de contenido

12 columnas, gap `space-4` (14px). Patrones: fila de 4 KPIs (col × 4), bloque métrica+chart (2/3 + 1/3), parejas label-valor (2 col).

### 5.4 Topbar de prototipo (solo en Claude Design)

Tira de 48px en la parte superior con:
- **Izquierda:** Toggle rol "Admin / User" + selector de estado de pantalla (default / loading / empty / error)
- **Derecha:** Avatar usuario (menú dummy con perfil y logout)

Esta topbar **no irá en producción**. Es un control de prototipo para que el cliente y el equipo técnico naveguen estados sin recargar.

Visualmente: fondo `--bg`, border-bottom `--line-2`. Altura 48px. Padding lateral `space-6`.

## 6. Componentes base

Como fondo y tarjeta son ambos blancos, **cada componente flotante necesita borde** (`--line`) para existir visualmente.

|---|---|---|

| subtle | Terciarias zonas densas | Bg `--line-2`, color `--ink`, sin borde |
| destructive | Eliminar, descartar | Bg `--neg`, color blanco |
| link | Navegación inline | Color `--accent-ink-deep`, subrayado en hover |
| icon | Solo icono | 32×32px, hover bg `--line-2` |

### 6.4 Select / Dropdown

Trigger igual que input. Contenido: bg `--card`, borde `--line`, sombra `shadow-e2`, radio `--radius-md`, padding 4px. Items: padding 8px 12px, hover bg `--accent-soft`, activo color `--accent-ink-deep`.

### 6.5 Checkbox y Radio

16-18px. Borde 1px `--muted-2`, radio `--radius-xs` (checkbox). Checked: bg `--accent`, check `--accent-ink`. Focus: `--shadow-focus`.

### 6.6 Toggle (Switch)

Track 36×20px, thumb 16×16px blanco. Off: track `--paused-bg`. On: track `--accent`. Uso: toggles de `show_campañas_*` en V5, toggle rol en topbar prototipo.

### 6.7 Slider

Track 4px `--line`, activo `--accent`, thumb 16px blanco con borde 2px `--accent`. Marca en valores enteros. Uso: filtro de valoración mínima en V3.

### 6.8 Cards

```css

  background: var(--card);
  border: 1px solid var(--line);
```

## 9. Reglas

### Sí
- Un solo elemento con acento sólido por vista.
- Borde `--line` en toda superficie flotante.
- Tabular-nums en todo número.
- Outfit con tracking negativo.
- Skeleton screens en loading.
- Empty states diseñados.
- Iconoir en todos los iconos.

### No
- Más de una `card-accent` por vista.
- Emojis como iconografía.
- Mezclar Iconoir con otra librería de iconos.
- Colores fuera de los tokens.
- shadcn/ui o Tailwind (este proyecto usa Bootstrap + DS).
- Texto sobre acento distinto a `--accent-ink`.
- Sombras dramáticas como sustituto del borde.
- Animaciones >240ms o con bounce.
- Responsive móvil.
- Spinners genéricos a pantalla completa.

## 10. Accesibilidad mínima (MVP)

- Contraste AA. Texto sobre acento siempre #282A2D.
- `:focus-visible` con `--shadow-focus` en todo interactivo.
- Teclado: Tab lógico, Esc cierra overlays, Enter activa primarios.
- Labels asociados a inputs. Roles ARIA en componentes custom.
- Iconos decorativos `aria-hidden`. Iconos accionables con `aria-label`.
- `<html lang="es">`.
